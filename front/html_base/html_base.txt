При разработке front части для веб-приложений используется слеующие технологии:
1) HTML - язык пердназначенный для описания структуры страницы
2) CSS - язык предназначенный для определения стилевых свойств (ширина,высота,цвет фона,цвет текста,тень и так далее)
HTML элементов
3) JavaScript- язык программироввания предназначенный для обработки событий возникающих в процессе работы пользователя со страаницей



Язык HTML строиться на использовании структуры тегов

HTML тэг - спциальная конструкция генерирующая на странице отдельно взятый элемент зависящий из используемого тега


HTML теги делятся на 2 вида :
1) Парные - тэг имеющий открывающую и закрывающую часть.Парные теги  могут
выступать контейнером для других HTML элементов
2) Не парные -  теги имеющие только открывающую часть.Не парные теги не могут выступать контейнером для
других HTML элементов


<tagName>...</tagName> - парный тэг с названием tagName где между откр и закрвающей частью могут быть описаны
внутренние HTML элементы для которых tagName будет выступать в качестве родителя.


<tagName> - не парный с названием tagName

Для каждого HTML тэга можно задавть несколько дополнительных аттрибутов

Аттрибут - некий конфигурационный параметр HTML элемента . Аттрибуты состоят из названия и значения
Аттрибуты всегда задаются в открывающей части HTML тега.

attr = "value" - атрибут attr со значением value.

<tagName attr = "value">...</tagName> - парный тэг с названием tagName и заданным аттрибутом attr со
значением value.


<tagName attr1 = "value1 " attr2 = "value2" ... </tagName>> - парный тэг с названием tagName  и заданными
аттрибутами attr1 и attr2 со значением value1 и value2 соответвственно

<!docType html> строка узакывающая на использование 5-й версии стандарта HTML

<html>...</html> - является базовым контейнером для всего содержимого странрицы.

<meta charset = "utf-8"> -назначает в качестве кодировки для страницы utf-8 которая позволит корректно отоброжать
любые символы.


<head> ... </head> - является контейнорм для мета-информации о страницы такой как ключевые слова слова для поиска,
описание,подключение дополнительных ресурсов (CSS файлы,JavaScript файлы,иконки и так далее).

<body> ... </body> - является контейнером для информации непорсредственно отбражаемой юзеру

По типу отображения элементов на страницы они подразделюяются на 2 основные группы:
1) Блочные - занимают все допустимое пространство по ширине родительского элемента за счет чего смещают свои соседние
элементы. Блочным элементам можно задавть размер (ширину и высоту) и отступы.
2) Линейные - занимают только то пространство которое определенно внутренним содержимым элемента за
счет чего выстраиваются по горизонтали друг за другом
Линейным элементам нельзя задавать размер и отступы

<p>...</p> - блочный текстовый контейнер

<span>...</span> -  линейный текстовый контейнер

<a>... </a> - ссылки при взаимодействии с которой производииться переход на другую страницу.
* href - адрес на который необходимо перейти после взаймодействия со ссылкой
* target = "_blank" - при наличии данного аттрибута и значения страница будет открыта в новой владке браузера


Списки в HTML создаются при помощи двух контейнеров тегов ul и ol


<ul>... </ul> - контейнер для создания маркироанного списка
<ol>... </ol> - контейнер для создания нумерованного списка

<li>...</li> - представляет собой одну запись списка


ul - unordered list  маркированный список
ol - ordered list нумеровванный список

li - list item  элемент списка


<table>...</table> - контейнер для формирования таблиц. HTML таблица состоит из рядов и ячеек

<thead>...</tbody> - заголовочная часть таблицы


<tbody>... </tbody> - информационная часть таблицы

<tr> ... </tr> -  ряд таблицы
<th> ... </th> -  заголовочная ячейка таблицы (жирный и центрированный текст)


<td> ... </td> -  инф ячейка таблицы

Ячейки созданные через th и td можно обьеденить  по горизонтали либо вертикали  в зависимости
от используемого аттрибута

* colspan - определяет количество ячеек по горизонтали которое должна занимать исходная ячейка

* rowspan - определяет количество ячеек по вертикали которое должна занимать исходная ячейка
